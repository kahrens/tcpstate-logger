# tcpstate-logger

An eBPF-based TCP state transition logger that emits structured OpenTelemetry log records to an OTLP gRPC receiver (OTel Collector or Loki).

## What it does

Attaches a kernel eBPF probe to the `sock/inet_sock_set_state` tracepoint. Every time a TCP socket transitions state (e.g. `TCP_CLOSE → TCP_SYN_SENT → TCP_ESTABLISHED → TCP_FIN_WAIT1 → TCP_CLOSE`), an event is sent from the kernel to userspace via a ring buffer. The Go userspace program reads those events and exports them as OTel log records over OTLP gRPC.

## Prerequisites

- Linux kernel 4.16+ with BPF and the `sock/inet_sock_set_state` tracepoint compiled in
- Clang/LLVM (tested with 20.x) — for compiling the eBPF C code
- Go 1.23+
- Root or `CAP_BPF`+`CAP_PERFMON` to load BPF programs

On WSL2 the tracepoint exists but tracefs is only readable as root. The binary must run as root.

## Build

```bash
# Compile eBPF C → object + generate Go bindings, then build the binary
make

# Or step by step:
go generate ./...   # compiles bpf/tcpstate.c → bpf_bpfel.{go,o} + bpf_bpfeb.{go,o}
go build -o tcpstate-logger .
```

`go generate` invokes `bpf2go` (from `github.com/cilium/ebpf/cmd/bpf2go`) via `go run` — no separate tool installation needed.

## Run

```bash
# Debug mode (stdout only, no collector required)
sudo ./tcpstate-logger --stdout

# Export to a local OTel Collector on port 4317
sudo ./tcpstate-logger --endpoint localhost:4317 --insecure --stdout

# Export directly to Loki (v2.9+ with OTLP gRPC enabled, default port 3100)
sudo ./tcpstate-logger --endpoint localhost:3100 --insecure --stdout
```

### Flags

| Flag | Default | Description |
|---|---|---|
| `--endpoint` | `localhost:4317` | OTLP gRPC target (`host:port`) |
| `--insecure` | `true` | Disable TLS (use for localhost) |
| `--service-name` | `tcpstate-logger` | OTel `service.name` resource attribute |
| `--stdout` | `false` | Also print events to terminal |

├── bpf_bpfeb.go            # generated — Go bindings for big-endian

├── bpf_bpfeb.o             # generated — compiled eBPF object

The `.go` and `.o` files under `bpf_bpf*` are generated artifacts — do not edit them. They are regenerated by `go generate ./...` whenever `bpf/tcpstate.c` changes.

## Architecture

### Kernel side: `bpf/tcpstate.c`

Hooks `SEC("tracepoint/sock/inet_sock_set_state")`. The tracepoint fires for all address families and protocols; the probe filters to `IPPROTO_TCP` only.

Per-event data collected (72-byte `struct event`):
- `timestamp_ns` — `bpf_ktime_get_ns()`
- `pid` — upper 32 bits of `bpf_get_current_pid_tgid()`
- `sport` / `dport` — source and destination ports
- `family` — `AF_INET` (2) or `AF_INET6` (10)
- `oldstate` / `newstate` — TCP state numbers (1–12)
- `saddr[16]` / `daddr[16]` — source/dest address; IPv4 uses first 4 bytes
- `comm[16]` — process name from `bpf_get_current_comm()`

Events are pushed via `BPF_MAP_TYPE_RINGBUF` (16 MB). If the ring is full, events are dropped silently.

The tracepoint context struct (`struct trace_event_inet_sock_set_state`) mirrors the kernel format at `/sys/kernel/tracing/events/sock/inet_sock_set_state/format` exactly. The 8-byte common header is mapped as `__u64 pad`.

### Userspace side: `main.go` + `otel.go`

`main.go` startup sequence:
1. `rlimit.RemoveMemlock()` — unlock memory for BPF map allocation
2. `setupOTel()` — create OTLP gRPC log exporter and batch processor
3. `loadBpfObjects()` — load the compiled eBPF bytecode (embedded in the binary by bpf2go)
4. `link.Tracepoint("sock", "inet_sock_set_state", ...)` — attach the probe
5. `ringbuf.NewReader(objs.Events)` — open the ring buffer
6. Read loop: `binary.NativeEndian` decode → `emitTCPStateEvent()`

`otel.go` maps each event to an `otellog.Record`:
- **Body**: `curl[1234] 192.168.1.1:55000 -> 93.184.216.34:443  TCP_CLOSE -> TCP_SYN_SENT`
- **Timestamp**: from `bpf_ktime_get_ns()` (monotonic boot clock)
- **Severity**: INFO
- **Attributes** (OTel semantic conventions):
  - `network.local.address` / `network.local.port`
  - `network.peer.address` / `network.peer.port`
  - `network.transport` = `"tcp"`
  - `tcp.state.old` / `tcp.state.new`
  - `process.pid` / `process.executable.name`

Records are batched and flushed via `sdklog.NewBatchProcessor`. On shutdown (SIGINT/SIGTERM), the ring buffer is closed and the OTel provider is flushed before exit.

## Generated type notes

bpf2go generates `bpfEvent` from `struct event` in `tcpstate.c`:
- `Comm` field is `[16]int8` (bpf2go maps C `char` → Go `int8`); `commString()` in `otel.go` handles the cast
- Field names are PascalCase from the snake_case C names: `timestamp_ns` → `TimestampNs`, `oldstate` → `Oldstate`
- The struct is 72 bytes with no implicit padding; `binary.NativeEndian` reads it directly

## Loki setup note

Loki v2.9+ accepts OTLP natively over gRPC. Enable it in `loki.yaml`:

```yaml
limits_config:
  allow_structured_metadata: true
otlp_config:
  resource_attributes:
    attributes_config:
      - action: index_label
        attributes:
          - service.name
```

Then point the logger at Loki's gRPC port (default 3100):
```bash
sudo ./tcpstate-logger --endpoint localhost:3100 --insecure
```
